KEY ARCHITECTURAL DECISIONS
================================================================================

1. STRATEGY PATTERN (Open/Closed Principle)
   - Allows adding new payment methods without modifying existing code
   - Dynamic strategy discovery via reflection
   - Polymorphic execution of charges
   
2. SOLID QUEUE (Background Jobs)
   - Zero external dependencies (no Redis)
   - Built-in recurring jobs
   - Automatic retry (3x, 10min interval)
   
3. BILLING GUARANTEES
   - No duplicate charges in same month
   - Automatic retry on failures
   - Isolated job execution (one client failure doesn't affect others)
